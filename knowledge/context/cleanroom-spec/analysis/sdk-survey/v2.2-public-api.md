# SDK survey — `vrops-adapters-sdk-2.2.jar`

**Source**: `inputs/from-devel/sdk/vrops-adapters-sdk-2.2.jar` (Navani's
devel pull, 2026-05-15)
**Built**: Apache Maven 3.3.9, OpenJDK 17.0.16, `Built-By: mts`
**Method**: `javap -p` against signatures only — no bytecode decompiled.
This is API-surface enumeration, not implementation analysis.

## Lineage

Two top-level package roots:

- `com.integrien.alive.common.adapter3.*` — the Integrien Alive
  framework (Integrien → vCenter Operations → vROps → Aria Operations →
  VCF Operations lineage). `adapter3` suggests this is the third
  generation of the adapter framework. ~470 classes; this is the bulk
  of the SDK.
- `com.vmware.vcops.common.*` and `com.vmware.vrops.logging.*` — newer
  VMware-namespaced additions (~10 classes). Mostly stats-provider
  helpers, localization, and logging.

## The contract: `AdapterInterface3`

Every native Java adapter implements this interface (typically by
extending `AdapterBase`). Nine methods:

| Method | Signature shape | Purpose |
|---|---|---|
| `configure` | `(AdapterConfig) → AdapterStatus` | Initial config + each subsequent config update |
| `describe` | `() → AdapterDescribe` | Return the adapter's model (combined with describe.xml) |
| `discover` | `(DiscoveryParam) → DiscoveryResult` | Auto-discover resources |
| `collect` | `() → CollectResult` | Periodic data collection (called every monitoring interval) |
| `test` | `(TestParam) → boolean` | Connectivity test (UI "Test Connection") |
| `discard` | `() → AdapterStatus` | Clean shutdown |
| `changePassword` | `(PasswordParam) → boolean` | Credential rotation |
| `checkCertificate` | `(CheckCertificateParam) → CheckCertificateResult` | TLS cert validation |
| `stopCollection` | `()` (default method) | Graceful stop of in-flight collect |

## The base class: `AdapterBase` (abstract, implements `AdapterInterface3`)

Uses the **template-method pattern**. The 9 interface methods are
declared `final` in AdapterBase (or near-final with finalizing logic);
they internally call `on*()` hooks that subclasses override.

### Required overrides (abstract in AdapterBase)

- `onConfigure(ResourceStatus, ResourceConfig)` — per-resource configure
- `onDescribe() → AdapterDescribe` — return the model
- `onDiscover(DiscoveryParam) → DiscoveryResult`
- `onCollect(ResourceConfig, Collection<ResourceConfig>)` — the heart

### Optional overrides (default impl in AdapterBase)

- `onConfigure(AdapterStatus, Collection<ResourceConfig>)` — batched variant
- `onStopResources(AdapterStatus, Collection<ResourceConfig>)`
- `onRemoveResources(AdapterStatus, Collection<ResourceConfig>)`
- `onChangePassword(PasswordParam) → boolean`
- `onTest(TestParam) → boolean`
- `onStopCollection()`
- `onDiscard()`
- `onCheckCertificate(CheckCertificateParam) → CheckCertificateResult`

The base class hands subclasses these helpers:

**Resource state**:
- `setResourceDown` (5 overloads, with/without LocalizedMsg and timestamp)

- `setResourceStatus(ResourceConfig, CommonConstants$ResourceStatusEnum, …)`

**Data push**:

- `addMetricData(ResourceConfig, List<MetricData>)`

- `addEvent(ResourceConfig, ExternalEvent)`

**Resource queries**:

- `getMonitoringResource(ResourceConfig | ResourceKey | Integer)`

**Metric collection-policy queries**:

- `isMetricMonitoring(…)` / `isMetricStopped(…)`

**Partial-collect support** (push data mid-collect, before `collect()` returns):
- `sendPartialDataToCollector()` (protected)

**TLS / connection**:
- `getKeyManagers()`, `getTrustManager()`, `getAdapterTrustManager()`,
  `getAdapterTrustStore()`, `getSocketFactory()`, `getVerifier()`

- `getCertificateMap()`, `getCertificateRenewalUrls()`
- static `getThumbprint(byte[])`

**Capability/extension registry**:
- `registerInterface(Class<?>, Object)` (final) and `queryInterface(Class<T>)` — runtime plug-in slot

**Adapter metadata**:
- `getAdapterName()`, `getAdapterFullName()`, `getAdapterKind()`
- `getAdapterConfig()`, `getAdapterInstResource()`, `getAdapterInstResourceId()`
- `getMonitoringInterval()`, `getMonitoringIntervalInSec()`

- `isDynamicMetricsAllowed()` — allow metrics not declared in describe.xml
- `isResourceRenameAllowed()` / `isResourceRenameAllowed(ResourceKey)`
- `isAutoCalculateRatedMetrics()` (protected)

**State**:
- `isDiscarded()` (post-discard guard)

## Optional opt-in interfaces (mix-ins)

Implement these alongside `AdapterInterface3` for extra capabilities:

- **`ActionableAdapterInterface`** (in `.action.`) — declare the adapter
  supports actions (operations against the monitored system from the UI
  or REST). Two methods: `onAction(ActionParam) → ActionResult`,
  `checkActionStatus(ActionResult, ActionParam) → ActionResult` (async).
  *Observed in: mpb-adapter*
- **`PartialCollectInterface`** — receive partial-collect callbacks via
  `processPartialData(AdapterBase, CollectResult)`.
- **`INonDisruptiveCertificate`** — handle unknown TLS certs without
  failing the collect. One method: `handleUnknownCertificate(UUID,
  InetSocketAddress, X509Certificate[]) → Collection<CertificateConfig>`.
- **`LicensableSolution`** — declare a license-bound solution.
  `decodeLicenseKey(String) → SolutionLicense`, `updateLicense(LicenseUpdateParam) → boolean`.
- **`AbstractDataFeedHandlerBase` / `DataFeedHandler`** — likely
  streaming push-data API (not yet inspected; flag for later).

## Open subsystems (flagged for future passes)

- **Action subsystem** (`adapter3.action.*`, 22 classes) — `ActionableAdapterInterface`, `ActionParam`, `ActionResult`, and supporting types. Will be observed concretely in mpb-adapter and likely vim.
- **Compatibility subsystem** (`adapter3.compatibility.*`, 14 classes) — version-shim layer. Unknown shape; revisit when SDK-version mismatches become relevant.
- **OpenAPI subsystem** (`adapter3.openapi.*`, 13 classes) — declarative REST API extension. Unknown shape.
- **Events subsystem** (`adapter3.events.*`, 14 classes) — `ExternalEvent` central; will see concrete usage in any adapter that emits events.
- **vcops stats provider** (`com.vmware.vcops.common.adapter3.statsprovider.*`, 1 class top-level) + `com.vmware.vcops.common.stats.*` (4 classes) — newer VMware add-on; unclear purpose, probably the bridge into Operations' main stats pipeline.
