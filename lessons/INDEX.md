Read before going down a path that looks obvious. If a lesson covers your
situation, heed it.

| [pak-content-localization-bundles.md](pak-content-localization-bundles.md) | Four localization bundles required for dashboard/view import. `localizationKey` in view XML must exactly match properties-file suffix — mismatch kills the entire content tree with `ERROR: Localization for key <x> is absent`. 2 builds (v21/v22) to isolate. |
| [heatmap-empty-groupby-crashes-renderer.md](heatmap-empty-groupby-crashes-renderer.md) | Heatmap `groupBy:{}` causes `JSONException("type not found")` in `HeatMapAction.initParam` — widget blank, Internal Server Error popup. | [content-root-is-content-dir.md](content-root-is-content-dir.md) | Authors wrote a dashboard + custom groups to the repo root instead of `content/`; loaders scan only `content/` so they validated "clean" but were invisible and unshippable. Stale agent prompts named bare paths post-migration. Repo-root `dashboards/` is live SDK-adapter pak content — don't delete it. |
| [sdk-paks-independent-repos.md](sdk-paks-independent-repos.md) | SDK paks are per-repo, gitignored, cloned via `context/managed_paks.md`. The official `.pak` is the deterministic CI build on a `v*` tag — never a dev/agent build. The `sdk-buildkit` is cut only by `tooling` on a tag. Factory publish emits a pointer to `<remote>/releases/latest`, not a binary. |
| [sdk-constants-are-display-names.md](sdk-constants-are-display-names.md) | `CommonConstants.VCOPS` (and all similar-looking SDK fields) are product display-name strings, not filesystem paths. `VCOPS = "VCF Ops"`. Never derive a file path from them. Use the empirically proven hard-wired default for `maintenanceuser.properties`. Live bug: compliance build 43, 2026-06-10. |
| [controller-describe-bare-instantiation.md](controller-describe-bare-instantiation.md) | During pak install the controller instantiates the adapter bare (no-arg reflection) and calls `describe()` with no platform injection — `getAdapterKind()` returns null. Never call injected-state accessors in the describe path. Supply the adapter kind key via `super(ADAPTER_KIND)` constructor. Live bug: compliance build 44, 3/3 install failures, controller rolled back to 43. |
| [suite-api-stitch-ssl-tofu-vs-java-http.md](suite-api-stitch-ssl-tofu-vs-java-http.md) | `CustomTrustManager` (platform TOFU manager) always throws on first contact with any unknown cert when used via `java.net.http.HttpClient`. There is no intercept/retry — `SSLHandshakeException` every cycle. Use `insecureSslContext()` (trust-all) for localhost Suite API calls; `getPlatformSslContext()` only for target-system connections where the admin has pre-approved the cert. Live bug: compliance build 45, every cycle 05:39Z+. |
| [setrelationships-foreign-adapter-scoped.md](setrelationships-foreign-adapter-scoped.md) | `setRelationships` emitted by a non-owning adapter onto a foreign resource is scoped per-reporting-adapter on VCF Ops 9.0.2 — it does NOT clobber the owning adapter's edges. Proven: synology build 16, devel 9.0.2, 2026-06-10 (wld01 iSCSI Datastore retained 22 VMWARE children + gained LUN child). No delta/labeled workaround needed. 9.1 unverified. |
| [pak-uninstall-cascades-credentials.md](pak-uninstall-cascades-credentials.md) | Pak uninstall on VCF Ops cascades into credential deletion — credentials return 404 after uninstall. Pre-flight: stash a credential manifest before DELETEing adapter instances; refuse if values aren't recoverable. |
| [no-volatile-status-in-reference-docs.md](no-volatile-status-in-reference-docs.md) | Build numbers / phase status / counts embedded in architecture/reference docs go stale (tier2_architecture.md "Phase 2 IN PROGRESS build 1.0.0.7"). Reference docs hold durable facts; volatile status lives in the SDK reviews / session-handoff or is derived. Prevention beats the curator's STALE-FACT detection. |
| [cross-mp-foreign-key-uniqueness-flags.md](cross-mp-foreign-key-uniqueness-flags.md) | A foreign `ResourceKey` for a cross-MP relationship/attachment must carry the resource's real uniqueness-bearing identifier set (propagate each `isPartOfUniqueness`). Hardcoding all-unique makes the key un-bindable: the edge is emitted every cycle, processed, never persisted — zero log trace. Direction is a non-constraint. Synology .18–.21 silent-drop → fixed 1.0.0.19. |
| [defect-registry-edits-are-corpus-coupled.md](defect-registry-edits-are-corpus-coupled.md) | Editing `context/defects.md` (even just `open→closed`) can turn real-corpus tests in `tests/test_defect_gate.py` red — change it via a PR, never a direct push to `main`, and update the coupled tests in the same change. DEF-001 closure broke 10 tests on `main`. |
